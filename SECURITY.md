# Security Policy

## Supported Versions

Proyecto activo en desarrollo. Se atienden reportes de seguridad sobre la rama principal.

## Reporting a Vulnerability

Si encuentras una vulnerabilidad:

1. No publiques el exploit en issues publicos.
2. Abre un reporte privado por GitHub Security Advisory o contacta al maintainer.
3. Incluye pasos de reproduccion y alcance.

Intentaremos confirmar recepcion rapidamente y comunicar un plan de correccion.
